# Psyche Injection — does telling the model *who you are* beat telling it *what's relevant*?

An open experiment from the [Glia](../..) project.

## The hypothesis

As context windows grow, the binding constraint on an LLM's usefulness shifts
from *retrieval* to *identity*. Systems like [gbrain](https://github.com/garrytan/gbrain)
are good at injecting **relevant context** — the pages that bear on your
question. This experiment tests a different, additive lever: injecting your
**psyche** — a map of who you are (values, worldview, essays, how you work) —
independent of the specific question.

The claim: an answer generated by a model that *knows who you are* serves you
better than one that only knows *what's relevant*, and best of all is a model
given both.

## Design — a 4-arm blind eval

For each task, one answer is generated under each condition:

| Arm | Context given to the model |
|---|---|
| **naked** | the prompt alone |
| **context** | prompt + relevant pages retrieved from the brain (what gbrain does) |
| **psyche** | prompt + a map of who the user is (what Glia's Context Bundle exports) |
| **best** | prompt + relevant context **+** psyche |

Then **3 independent judges**, each given the user's psyche as *ground truth*,
**blind-rank** the four answers (shuffled, de-identified — the judge does not
know how any was produced) on: specificity, actionability, personal fit, and
genuine insight. Answer→slot assignment is permuted per (task, judge) to
cancel position bias.

### Controls & threats to validity
- **Negative control:** one task is purely technical (graph algorithms), where
  the user's brain holds nothing relevant and psyche *should not* help. If
  psyche wins there, the judge is biased toward mirroring — and we say so.
- **Mirroring risk:** the judge sees the psyche and could over-reward answers
  that echo it. Mitigated by rubric dimensions that separate *personal fit*
  from *insight/usefulness*, and by reporting them separately.
- **Self-preference:** generator and judge share a model family here; a
  cross-family judge would strengthen the result. Noted, not yet done.
- **n is small** (6 tasks × 3 judges). This is a signal-finding pilot, not a
  publication-grade benchmark — the harness is built to scale up.

## Run it yourself

Materials (the psyche + per-task context) are built from a real brain and are
**gitignored** (private). The harness, tasks, and methodology are public.

1. Build materials: a psyche map (e.g. `glia` → Export Context → identity) at
   `materials/psyche.md`, tasks at `materials/tasks.json`, and per-task
   relevant context at `materials/<id>-context.md`.
2. Run the workflow (`psyche-injection-eval` — see the script the harness
   emits) to generate + judge.
3. `python3 aggregate.py results/raw/results.json` → `REPORT.md`.
