# Psyche Injection — does telling the model *who you are* beat telling it *what's relevant*?

An open experiment from the [Glia](../..) project.

## The hypothesis

As context windows grow, the binding constraint on an LLM's usefulness shifts
from *retrieval* to *identity*. Systems like [gbrain](https://github.com/garrytan/gbrain)
are good at injecting **relevant context** — the pages that bear on your
question. This experiment tests a different, additive lever: injecting your
**psyche** — a map of who you are (values, worldview, essays, how you work) —
independent of the specific question.

The claim: an answer generated by a model that *knows who you are* serves you
better than one that only knows *what's relevant*, and best of all is a model
given both.

## Design — a 4-arm eval, run twice

For each task, one answer is generated under each condition:

| Arm | Context given to the model |
|---|---|
| **naked** | the prompt alone |
| **context** | prompt + relevant pages retrieved from the brain (what gbrain does) |
| **psyche** | prompt + a map of who the user is (what Glia's Context Bundle exports) |
| **best** | prompt + relevant context **+** psyche |

Independent judges blind-rank the four answers (shuffled, de-identified — the
judge doesn't know how any was produced), with answer→slot permuted per
(task, judge) to cancel position bias. We ran this **twice**, and the
difference between the runs is the whole point:

- **v1 (signal-finding):** 6 tasks × 3 judges. Judges were given the psyche as
  ground truth and scored a **personal-fit** dimension.
- **v2 (mechanism isolation):** 9 tasks × 2 judges, then **v3** re-judges the
  same answers with 5 more judges each (→ 7/task). Judges are **blind to the
  psyche**, the personal-fit dimension is **removed**, and two neutral tasks are
  scored **objectively** against a fixed rubric (one verified by executing the
  answer's code).

**Why twice:** v1 produced a striking headline. Two things could have faked it —
judges rewarding a recognizable voice, and a rubric that paid out for sounding
personal. v2/v3 remove both. A result worth your name is one you've tried, in
public, to kill.

### Threats to validity (and what each run does about them)
- **Mirroring / voice bias** — v1's judges could see the psyche. **v2 fixes
  this:** judges are blind and there's no personal-fit dimension. (This is what
  overturned v1's "psyche alone dominates.")
- **"Model just tries harder"** — priming with a person might lift *everything*.
  **v2 fixes this:** neutral technical controls, objectively scored — all arms
  tie at ceiling, so the identity lift is task-specific, not global.
- **Length confound** — blind judges might reward longer answers. Checked: all
  arms average ~2.2k chars; not the driver.
- **Judge noise** — v2 had only 2 judges/task. **v3 fixes this:** 7 judges/task
  on the same fixed answers.
- **Self-preference** — generator and judge were both Opus. **v4** re-judged with
  Haiku 4.5 (still Anthropic) and reproduced the ordering; **v6 closes it across
  VENDORS:** OpenAI's **gpt-5**, blind and position-bias-controlled, reproduces
  best-first (beats context 63%, psyche 67%) and the specificity/insight mechanism
  — so it isn't Anthropic grading its own homework. Nuance: gpt-5 is harsher on
  psyche-*alone* (tail order is judge-dependent), and it penalizes the identity
  answers' *real* specifics as "fabrication" because it can't verify them — meaning
  blind judging **understates** identity (the numbers are a floor). gpt-4o was
  unusable (90% position bias). See [REPORT-v6.md](REPORT-v6.md).
- **n is still small.** Signal-finding pilot, not a publication benchmark — the
  harness scales.

## Run it yourself

Materials (the psyche + per-task context) are built from a real brain and are
**gitignored** (private). The harness, tasks, and methodology are public.

1. Build materials: a psyche map (e.g. `glia` → Export Context → identity) at
   `materials/psyche.md`, tasks at `materials/tasks-v2.json`, and per-task
   relevant context at `materials/<id>-context.md`.
2. Generate + blind-judge: run `harness/eval-v2.js` (the workflow the harness
   emits). To tighten confidence without regenerating, `harness/eval-v3-rejudge.js`
   re-judges the saved answers with more blind judges.
3. Aggregate: `python3 aggregate-v2.py results/raw/results-v2.json` → `REPORT-v2.md`,
   and `python3 aggregate-v3.py` to merge v2+v3 → `REPORT-v3.md`.

- [VERBOSITY-CHECK.md](VERBOSITY-CHECK.md) — robustness: the lift isn't a length

- [SIGNIFICANCE.md](SIGNIFICANCE.md) — task-clustered uncertainty: direction is
  robust (6/7 tasks, 3 vendors), but n=7 limits significance — ordering is solid,
  exact percentages are directional.

**Honest headline (v2/v3, blind):** injecting **who you are** makes an agent
measurably sharper — but as a **complement** to what's relevant, not a
replacement. The **both** arm wins the blind ranking and beats relevance-alone
71% (6/7 tasks); identity's specific contribution is **insight** (retrieval buys
specificity and actionability). On neutral technical tasks every arm ties at
ceiling, so the lift is specific to identity-shaped work, not a global "try
harder" effect. The **ordering** is robust — it reproduces across three judge
vendors (Opus, Haiku, gpt-5) and survives a verbosity control — but this is a
**7-task pilot**: trust the ordering, read the exact percentages as directional
(see [SIGNIFICANCE.md](SIGNIFICANCE.md) / [VERBOSITY-CHECK.md](VERBOSITY-CHECK.md)).

**What v1 got wrong (kept in the repo on purpose):** the first, *non-blind* run
claimed psyche *alone* beat the combination and that retrieval *diluted*
identity. That didn't survive blind judging — it was mostly judges rewarding a
voice they could see. The correction is the finding.
